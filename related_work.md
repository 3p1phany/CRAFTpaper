## 7. Related Work

**Row buffer management with static policies and address mapping.**
The open-page and close-page tradeoff and the limitations of existing adaptive schemes are discussed in Section 2. Quantitative comparisons with ABP, DYMPL, and INTAP are presented in Section 5. JEDEC[DDR+345] standards define a per-command auto-precharge mechanism that appends a precharge directive to individual read or write commands, upon which all adaptive row buffer management schemes are constructed. Kaseridis et al. [Kaseridis+11] proposed Minimalist Open-page, which limits the number of row buffer hits per activation through an address mapping that distributes column address bits across the bank address field. This address mapping balances row buffer locality against bank-level parallelism. However, the fixed hit limit is determined at design time and does not adapt across workloads or execution phases. Mi et al. [Mi+09] proposed PARBLO, a software-based approach that controls operating system page allocation order to improve data placement in DRAM and reduce row buffer conflicts. PARBLO is complementary to hardware-based page policy mechanisms, though its applicability is restricted to programs with regular, affine loop structures. Unlike static policies or fixed address mappings, CRAFT continuously adjusts the idle timeout per bank, enabling it to track workload phase changes and inter-bank locality differences at runtime.

**Predictor-based row buffer management.**
 Park and Park [Park+03] maintained per-bank or per-row two-bit saturating counters that track page hit and miss history to predict whether the next access will hit the currently active row, thereby deciding whether to keep the row open or issue an auto-precharge command. Their per-bank variant requires only four counters, while the per-row variant demands storage that scales with memory capacity. Stankovic and Milenkovic [Stankovic+05] proposed a cascaded close-page predictor that decomposes the precharge decision into two stages. A zero-live-time predictor uses per-row two-bit saturating counters to identify rows that receive only a single access per activation and issues an immediate auto-precharge for such rows. For rows predicted to have subsequent accesses, a dead-time predictor monitors the elapsed idle time and triggers precharge when the idle period exceeds twice the most recent access interval. Their analysis noted that incorrectly closing a row that will be reaccessed is costlier than incorrectly keeping a row open, since the latter can still be corrected by the dead-time predictor. However, this asymmetry is not exploited to drive its adaptive page policy adjustment. CRAFT derives asymmetric adjustment magnitudes directly from DRAM timing parameters and applies them as the feedback signal for continuous timeout adaptation. Xu et al. [Xu+09] adapted two-level branch prediction to page policy by maintaining a history register and a pattern history table with two-bit saturating counters. They explored per-global, per-bank, and per-page granularities for both the history register and the pattern history table. The per-page variants achieve the highest prediction accuracy but require storage that grows linearly with the number of tracked pages. A fundamental challenge shared by these predictor-based approaches is that memory requests have already been filtered through multiple cache levels before reaching the memory controller, weakening the assumption that these memory requests exhibit the same predictable correlations as branch instruction streams.

Awasthi et al. [Awasthi+11] proposed ABP. ABP maintains a per-row history table to predict the total access count for each row and closes the row buffer after the predicted count is reached. Its storage overhead and per-row granularity limitations are detailed in Section 2. The predictor lookup itself is hidden from the critical path. However, the fixed-capacity history table additionally suffers increasing conflict and capacity misses with growing bank counts per channel. This effect degrades prediction accuracy for workloads with large active page sets. Shen et al. [Shen+14] proposed RBPP, which maintains a small set of per-bank registers storing the addresses of frequently accessed rows. If the currently active row matches one of these registered entries, the row is kept open; otherwise it is closed immediately. RBPP reduces storage overhead by 91% compared to ABP. However, its saturating counters cause registered hot rows to persist in the tracker across phase changes, slowing adaptation to shifting program phases. FAPS-3D [Rafique+19] deploys per-bank two-bit saturating counter state machines within 3D-stacked DRAM vault controllers to classify banks into four hit-rate tiers and switch between open-page and close-page policies accordingly. However, its reliance on vault controllers confines its applicability to 3D-stacked DRAM architectures.

**Timeout-based adaptive schemes.**
Timeout-based approaches precharge a row after a configurable idle period following the last row-hit command. By adjusting the timeout value, the policy spans the full range from open-page to close-page behavior. INTEL et al. [INTEL+06] proposed INTAP. INTAP accumulates precharge errors in a per-bank mistake counter and adjusts the timeout by a fixed step size. Its symmetric, cost-unaware adjustment mechanism is analyzed in Section 2. Beyond this fundamental limitation, the fixed step size constrains adaptation granularity and limits responsiveness to rapid program phase transitions.

**Learning-based approaches.**
Ipek et al. [Ipek+08] applied reinforcement learning to DRAM command scheduling. The approach requires 32 KB of SRAM for function approximation tables and a multi-stage pipelined learning engine operating at processor frequency. This implies substantial hardware complexity and practical deployment challenges. DYMPL [Rafique+22] models page policy as a binary classification problem using perceptron learning. Its feature set includes page utilization, hotness, recency, and column stride. The associated storage and critical-path computation overhead are detailed in Section 2. These learning-based approaches share a common reliance on indirect features as proxies for the optimal precharge decision. Such indirection inherently introduces estimation error and training overhead. CRAFT instead uses the precharge outcome itself as the sole feedback signal. This design eliminates the need for feature engineering, model training, or associative table lookups.

**Application-aware and cross-layer techniques.**
Xie et al. [Xie+13] combined bank partitioning with application-aware page policy assignment. Their approach profiles per-application memory access intensity and row buffer hit rate at runtime to assign open-page or close-page policy at bank-level. This approach requires bank-level isolation across applications and constrains memory address mapping flexibility. Rixner et al. [Rixner+00] introduced FR-FCFS scheduling. FR-FCFS reorders pending memory requests to prioritize row buffer hits. FR-FCFS addresses request ordering rather than the page policy decision and is orthogonal to CRAFT. Lee et al. [Lee+08] proposed prefetch-aware DRAM scheduling. This technique adaptively adjusts the scheduling priority of prefetch requests based on per-core prefetch accuracy measurements. Prefetch-aware scheduling is complementary to CRAFT's per-bank timeout mechanism.  Ghasempour et al. [Ghasempour+16] proposed HAPPY. HAPPY exploits the fixed mapping between physical address bits and DRAM address to replace per-row monitoring with per-address-bit monitoring. This encoding substantially reduces storage overhead and maintains comparable prediction accuracy. HAPPY is orthogonal to CRAFT's cost-asymmetric timeout adaptation. CRAFT's per-bank state of 35 bits per bank already achieves minimal storage overhead. The storage compression benefit of per-address-bit encoding is therefore limited in CRAFT's context. Nevertheless, the two approaches address complementary aspects of the design space.

**DRAM architectural modifications.**
Several works have addressed row buffer performance through modifications to the DRAM device itself. Subramanian et al. [Subramanian+18b] introduced isolation and equalization transistors. These transistors decouple the sense amplifier from the bitline and enable simultaneous read and precharge operations. This design reduces effective tRCD by 25%. Rohbani et al. [Rohbani+21] proposed PF-DRAM. PF-DRAM eliminates the precharge phase entirely through an alternative bitline sensing circuit. This circuit handles all possible initial bitline voltage states without requiring a conventional precharge step. Gulur et al. [Gulur+12] proposed replacing the single monolithic row buffer with multiple smaller sub-row buffers to capture temporal locality across multiple rows simultaneously. This design enables early precharge scheduling and intra-bank parallelism. These circuit-level techniques require changes to the DRAM chip. CRAFT operates entirely at the memory controller level and requires no modifications to the DRAM device. The two categories of approach are orthogonal. CRAFT's adaptive timeout mechanism can be combined with these architectural modifications for compounded latency and energy improvements.
